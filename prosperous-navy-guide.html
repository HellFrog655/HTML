แผนผัง 05
SOP — ขั้นตอนปฏิบัติมาตรฐาน
Activity Diagrams
การไหลข้อมูล ▼
📌
⚓ เปรียบกับ
SOP — ระเบียบปฏิบัติประจำบนเรือ
เช่น SOP ดับเพลิงบนเรือ — กดสัญญาณเตือนก่อน → ปิดระบบน้ำมัน → ฉีดน้ำ → ถ้าดับได้: รายงาน → ถ้าดับไม่ได้: อพยพ ทุกขั้นตอนมีลำดับที่ต้องทำตามเป๊ะ ถ้าข้ามขั้นอาจเกิดความเสียหาย
SOP 5 ฉบับสำหรับระบบนี้
①
SOP ส่งฟอร์มขอใบเสนอราคา
เปิดหน้าสินค้า → กรอกชื่อ-เบอร์-บริษัท → กดส่ง → ระบบตรวจสอบ → ✓ ครบ: บันทึก + ส่งอีเมล Admin + แสดงยืนยัน / ✗ ไม่ครบ: แจ้ง error ให้กรอกใหม่
②
SOP สมัครหลักสูตร
เลือกหลักสูตร → กรอกข้อมูล → ตรวจที่นั่งว่าง → ✓ ว่าง: ยืนยัน / ✗ เต็ม: แจ้งรออบรมรอบหน้า
③
SOP Admin Login
กรอก username + password → ตรวจสอบ → ✓ ถูก: ออกบัตรดิจิทัล เข้า Dashboard / ✗ ผิด 5 ครั้ง: ล็อค 15 นาที
④
SOP จัดการ Lead
เปิดรายการ → เลือก Lead → โทรหา → กดเปลี่ยนสถานะ "ติดต่อแล้ว" → ติดตามต่อ หรือ ปิดงาน
⑤
SOP เพิ่มบทความ
กดเพิ่ม → เขียน → บันทึกร่าง → ตรวจทาน → เผยแพร่ → ขึ้นเว็บทันที
⚓
สรุปสั้น: ทำไมเรือรบต้องมี SOP? เพราะ ในสถานการณ์กดดันทุกคนต้องทำถูกต้องโดยไม่ต้องคิดใหม่ ระบบเว็บก็เช่นกัน ทุก action มีขั้นตอนที่ถูกต้องกำหนดไว้ล่วงหน้า

บันทึกการสื่อสารทางวิทยุ
Sequence Diagrams
การไหลข้อมูล ▼
📻
⚓ เปรียบกับ
บันทึกการสื่อสารวิทยุระหว่างเรือ
เรือ A: "ALPHA เรียก BRAVO รายงานตำแหน่ง Over" → เรือ B: "BRAVO ที่ตำแหน่ง X Over" → เรือ A: "รับทราบ Out" — บันทึกชัดว่า ใครพูดก่อน พูดอะไร อีกฝ่ายตอบว่าอะไร ตามลำดับเวลาเป๊ะ
3 บทสนทนาหลักของระบบ
①
ลูกค้าส่งฟอร์ม (7 รอบสื่อสาร)
ลูกค้า → กดส่ง → หน้าเว็บตรวจ → ส่งไป Server → Server บันทึกลงฐานข้อมูล → ฐานข้อมูลรับทราบ → Server ตอบหน้าเว็บ → แสดงยืนยันให้ลูกค้า
②
Admin Login (8 รอบสื่อสาร)
Admin → กรอกรหัส → ส่งไป Server → Server ดึงรหัสเข้ารหัสจาก DB → เปรียบเทียบ → ถูก: ออกบัตรดิจิทัล + เข้า Dashboard / ผิด: ส่ง error กลับ
③
สมัครหลักสูตร (6 รอบสื่อสาร)
ลูกค้า → สมัคร → Server ตรวจที่นั่ง → ว่าง: บันทึก + ส่งอีเมลยืนยัน / เต็ม: แจ้งกลับ
⚓
สรุปสั้น: ต่างจาก SOP ตรงที่แผนผังนี้ แยกให้เห็นว่าแต่ละหน่วยรับผิดชอบส่วนไหน — หน้าเว็บทำอะไร, Server ทำอะไร, ฐานข้อมูลทำอะไร เหมือนแยกภารกิจให้แต่ละเรือในกองเรือ

⚓ เปรียบกับ
DEFCON — ระดับความพร้อมรบ
DEFCON 5 (ปกติ) → DEFCON 3 (ยกระดับ) → DEFCON 1 (พร้อมรบเต็มที่) — ข้ามระดับทันทีไม่ได้ ต้องผ่านเงื่อนไขก่อน และบางระดับกลับไม่ได้ง่ายๆ

📝
บทความข่าว
"ร่าง" → (กดเผยแพร่) → "เผยแพร่" → (กดลบ) → "ลบแล้ว" · ลบแล้วกู้ไม่ได้ เหมือนคำสั่งที่ออกไปแล้ว
📥
Lead ผู้ติดต่อ
"ใหม่" → (โทรหา) → "ติดต่อแล้ว" → (ติดตาม) → "Follow up" → "ปิดงานได้ WIN" หรือ "ล้มเหลว LOSE" · ปิดแล้วย้อนไม่ได้
🎓
การสมัครหลักสูตร
"รอยืนยัน" → (Admin อนุมัติ, มีที่นั่ง) → "ยืนยัน" → (วันอบรมจบ) → "เสร็จสิ้น" · หรือยกเลิกได้ก่อนอบรม
🤿
ไลฟ์การ์ด
"ว่าง" → (รับงาน) → "ปฏิบัติการ" → (งานจบ) → "ว่าง" ใหม่ · หรือ → "อบรม" → กลับมา "ว่าง"
⚓
สรุปสั้น: เหมือนระเบียบขั้นตอนในกองทัพที่ ห้ามข้ามขั้น ห้ามถอยผิดลำดับ ป้องกันความผิดพลาดที่ทำให้ระบบสับสน เช่น Lead ยังไม่โทรแต่กดปิดงานแล้ว

Security Clearance ของกองทัพ

✅

❌
พลทหาร (ลูกค้า) เข้าไม่ได้ — ดูบทความร่าง, เห็น Leads คนอื่น, แก้ไขข้อมูลใดๆ

สรุปสั้น: Security Clearance ป้องกันไม่ให้ข้อมูลลับหลุดไปผิดคน — แผนผังนี้ ป้องกันไม่ให้ลูกค้าทั่วไปเห็นข้อมูลที่ Admin ควรเห็นคนเดียว

ระบบโลจิสติกส์ — โรงงาน → คลัง → หน้างาน
กระสุนผลิตที่โรงงาน → ส่งไปคลังสินค้ากลาง → คลังตรวจสอบคุณภาพ → ส่งไปหน้างานทันทีที่ได้รับการร้องขอ — มีระบบตรวจสอบทุกขั้น ไม่มีของหลุดถึงหน้างานแบบไม่ผ่านการตรวจ
ขั้นตอนส่งโค้ดขึ้นเว็บ
1️⃣
โรงงาน (Local) — นักพัฒนาเขียนโค้ดบนเครื่องตัวเอง ทดสอบเบื้องต้น เหมือนผลิตกระสุน
2️⃣
ส่งเข้าคลัง (GitHub) — โค้ดที่เขียนเสร็จส่งเข้าคลังกลาง ระบบตรวจสอบอัตโนมัติทันที
3️⃣
ตรวจสอบคุณภาพ (Preview) — สร้างเว็บทดสอบขึ้นมาให้ดูก่อน เจ้าของโปรเจกต์ตรวจแล้วสั่ง approve
4️⃣
ส่งหน้างาน (Production) — approve แล้ว → ขึ้น prosperous-rescue.com ทันที มี HTTPS อัตโนมัติ
⚓
สรุปสั้น: ระบบนี้ทำให้ อัปเดตเว็บได้โดยไม่ต้อง down ระบบ มีจุดตรวจสอบก่อนขึ้นจริงทุกครั้ง เหมือนกระสุนทุกนัดต้องผ่าน QC ก่อนส่งหน้างาน

📡
⚓ เปรียบกับ
คู่มือ NATO STANAG — ศัพท์มาตรฐานพันธมิตร
พันธมิตรหลายชาติต้องใช้คำเดียวกัน "ALPHA = เรือ A ไม่ใช่เรือ B" — ถ้าแต่ละคนใช้ศัพท์ต่างกัน ในสนามรบอาจยิงกันเองโดยไม่ตั้งใจ
ศัพท์ที่ทุกคนในโปรเจกต์ต้องรู้ตรงกัน

⚕️ ศัพท์ธุรกิจ Prosperous

  • AED — เครื่องช็อคหัวใจ (กดปุ่มเอง)
  • Auto AED — ช็อคอัตโนมัติ ไม่ต้องกด
  • CPR — ปั๊มหัวใจช่วยชีวิต
  • ILSE — มาตรฐานไลฟ์การ์ดสากล
  • Lead — คนที่ส่งฟอร์ม ยังไม่ซื้อ
  • CSR — กิจกรรมเพื่อสังคมองค์กร

💻 ศัพท์เทคนิค (แปลเป็นทหาร)

  • API — ช่องทางสื่อสารระหว่างหน่วย
  • JWT — บัตร Security Clearance ดิจิทัล
  • Supabase — คลังข้อมูลกลาง
  • Vercel — ฐานทัพที่เว็บอยู่
  • UUID — หมายเลขประจำตัวที่ไม่ซ้ำกันเลย
  • CRUD — สร้าง/ดู/แก้/ลบ (4 ภารกิจพื้นฐาน)
⚓
สรุปสั้น: ในกองทัพพูดว่า "TANGO" ทุกคนรู้ทันทีว่าหมายถึงศัตรู ไม่ต้องอธิบายใหม่ทุกครั้ง — Glossary ทำให้ ทุกคนในโปรเจกต์คุยภาษาเดียวกันโดยไม่ตีความผิดกัน
