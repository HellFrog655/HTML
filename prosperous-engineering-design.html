📋 Non-functional Requirements (NFR)

6 หมวด
⚡ Performance
  • LCP < 2.5 วินาที (Core Web Vitals)
  • TTI < 3.5 วินาที บน 4G
  • API response < 500ms (p95)
  • Contact form submit < 2 วินาที
🔒 Security
  • HTTPS ทุก endpoint (SSL/TLS 1.3)
  • JWT httpOnly cookie · bcrypt cost=12
  • Rate limiting: 10 req/min ต่อ IP
  • Input sanitize + CSRF protection
  • Admin session timeout 8 ชั่วโมง
🟢 Availability
  • Uptime SLA ≥ 99.9% (Vercel)
  • Zero-downtime deployment
  • CDN Edge caching (Asia/Singapore)
  • Auto-failover via Vercel platform
📈 Scalability
  • Serverless auto-scale (Vercel)
  • รองรับ concurrent users 500+
  • Supabase connection pooling
  • Static pages cached via CDN
♿ Accessibility & UX
  • รองรับ mobile (responsive, ≥ 375px)
  • รองรับภาษาไทย + Sarabun font
  • SEO: meta tags, OG, sitemap.xml
  • ไม่มีปุ่ม "ซื้อ" — B2B inquiry only
🛠 Maintainability
  • TypeScript strict mode ทุก component
  • CI/CD: GitHub → Vercel auto-deploy
  • Lint + build check ก่อน merge
  • Content update ผ่าน Admin CMS
  • Log: Vercel Function logs + analytics

🔐 Token Lifecycle
// Login
POST /api/auth/login → JWT issued
Set-Cookie: token=... HttpOnly Secure
Max-Age: 28800 (8 hours)

// Every Admin Request
middleware.ts: jwtVerify(cookie.token)
  → valid: continue to handler
  → expired: clear cookie + 401
  → invalid: 401 UNAUTHORIZED

// Logout
POST /api/auth/logout → Set-Cookie: token= Max-Age=0
StatusError Codeเกิดเมื่อAction ที่ควรทำ
400VALIDATION_ERRORfield ขาด / รูปแบบไม่ถูก / type ผิดแสดง field-level error messages
401UNAUTHORIZEDไม่มี JWT cookie / expired / invalid signatureredirect → /admin/login พร้อม message
403FORBIDDENมี JWT แต่ role ไม่พอ (superadmin only actions)แสดง "ไม่มีสิทธิ์" — ไม่ redirect
404RESOURCE_NOT_FOUNDUUID ไม่มีใน DB หรือ soft-deletedแสดง "ไม่พบ resource"
409CONFLICT / COURSE_FULL / SLUG_CONFLICT / DUPLICATE_EMAILที่นั่งเต็ม / slug ซ้ำ / email สมัครซ้ำแสดง specific message ตาม code
422BUSINESS_RULE_ERRORdata ถูก format แต่ business logic ไม่ผ่าน เช่น ILSE cert ไม่ครบแสดง reason ชัดเจน
429RATE_LIMIT_EXCEEDEDเกิน rate limit (contact: 3/min, login: 5/15min)แสดง "กรุณารอ X วินาที" + Retry-After header
500INTERNAL_SERVER_ERRORDB connection failed / unhandled exceptionlog ไปยัง Vercel Functions Log · แสดง generic message เท่านั้น
หลักการ Error Handling: ① Validate input ด้วย Zod schema ก่อน ② Check business rules ③ Try DB operation ④ On error: log server-side แต่ return generic message ไปยัง client — ห้าม leak stack trace หรือ DB details
